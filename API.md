# CineSound API

## Base URL

- **Local dev:** `http://localhost:8000`
- **Production:** the URL of your Railway deployment (see `DEPLOY.md`)

## Authentication

CineSound is **mostly unauthenticated**. Per-device identity is supplied
by the caller as a `session_id` string in the request body or query
parameter — the frontend stores this in a 1-year cookie generated by
`frontend/lib/session.ts::getOrCreateSessionId()`.

The only endpoint that *verifies* identity is `POST /signin`, which
takes a Google ID token and migrates the session's memory to a stable
`google:<sub>` user ID.

Production deployments protect cost (not access) via:

- **Per-IP rate limit** — 10 requests/hour on `/query` and `/playlist`
  (`slowapi`, in-memory, see `app/middleware/rate_limit.py`)
- **Daily LLM-call cap** — `DAILY_QUERY_CAP` env var; over the cap,
  `/query` returns the stub `CAP_REACHED_REC` and `/playlist` returns 429

> ⚠️ Assumption: no per-user auth tokens / API keys. If you need to
> restrict access entirely, add an auth dependency upstream.

## Endpoints

### `GET /health`

App + DB health check.

**Response (200):**
```json
{ "status": "ok", "env": "dev", "db": true }
```

**curl:**
```bash
curl http://localhost:8000/health
```

---

### `GET /health/db`

Confirms the asyncpg pool can `SELECT 1`.

**Response (200):**
```json
{ "status": "ok", "select_1": 1 }
```
Returns `{"status":"disabled"}` if no `DATABASE_URL` is configured.

**curl:**
```bash
curl http://localhost:8000/health/db
```

### `POST /query`

**Streams** the orchestrator's progress + final `Recommendation` via
Server-Sent Events. Rate-limited and cost-capped.

**Request body** (`QueryRequest` in `app/schemas.py`):
```json
{ "query": "string, 1-1000 chars", "session_id": "string, 1-128 chars" }
```

**Response:** `text/event-stream`. Event sequence:
```
event: ack
data: {"session_id":"..."}

event: node_done
data: {"node":"load_memory" | "profile" | "search" | "rank_and_pair" | "save_memory"}

event: final
data: <Recommendation JSON>
```
On orchestrator failure:
```
event: error
data: {"message":"..."}
```

**`Recommendation` schema** (final event payload):
```json
{
  "mood_detected": "string",
  "pairings": [
    {
      "movie": {
        "tmdb_id": 0,
        "title": "string",
        "year": 0,
        "rating": 0.0,
        "genres": ["string"],
        "reason": "string",
        "trailer_url": "string|null",
        "poster_url": "string|null"
      },
      "music": {
        "spotify_uri": "string",
        "track": "string",
        "artist": "string",
        "album": "string|null",
        "mood_tag": "string",
        "reason": "string",
        "spotify_url": "string",
        "album_art_url": "string|null",
        "preview_url": "string|null"
      },
      "pairing_note": "string"
    }
  ],
  "fallback_message": "string|null"
}
```

When daily cap is hit or no candidates survive filtering, `pairings` is
empty and `fallback_message` carries the user-facing text.

**curl:**
```bash
curl -N -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"query":"I just finished Interstellar, feeling reflective","session_id":"demo:1"}'
```

### `POST /feedback`

Captures a thumbs vote against either a movie or a track. Looks up the
target's genres in `embeddings` and appends to `liked_genres` or
`disliked_genres` for the session.

**Request body** (`Feedback` schema):
```json
{
  "session_id": "string, 1-128",
  "vote": "up" | "down",
  "tmdb_id": 0,                       // either tmdb_id...
  "spotify_uri": "spotify:track:..."  // ...or spotify_uri (one required)
}
```

**Response:** `204 No Content`

**Errors:** `400` if neither `tmdb_id` nor `spotify_uri` is provided.

**curl:**
```bash
curl -X POST http://localhost:8000/feedback \
  -H "Content-Type: application/json" \
  -d '{"session_id":"demo:1","vote":"down","tmdb_id":12345}'
```

### `POST /signin`

Verifies a Google OAuth ID token via the public tokeninfo endpoint
(`https://oauth2.googleapis.com/tokeninfo`), checks `aud` matches
`GOOGLE_CLIENT_ID`, and migrates the anonymous session's memory rows
to `google:<sub>`.

**Request body:**
```json
{ "id_token": "string (Google ID token)" }
```

**Headers / Cookies:** `session_id` cookie (the device's anonymous ID)

**Response (200):**
```json
{ "user_id": "google:<sub>", "migrated_keys": 0 }
```

**Errors:**
- `400` — missing `id_token`
- `401` — invalid token, audience mismatch, or missing `sub`
- `500` — `GOOGLE_CLIENT_ID` not configured on the server

**curl:**
```bash
curl -X POST http://localhost:8000/signin \
  -H "Content-Type: application/json" \
  -b "session_id=session:abc" \
  -d '{"id_token":"<google id token>"}'
```

### `GET /me`

Returns a summary of the session's memory + recent activity. Drives
the frontend's `TasteProfilePanel`.

**Query parameters:**
- `session_id` (required) — 1–128 chars

**Response (200):**
```json
{
  "session_id": "string",
  "counts": {
    "watched_movies": 0,
    "heard_tracks": 0,
    "queries_with_mood": 0
  },
  "top_liked_genres":    [{ "genre": "Sci-Fi", "count": 3 }],
  "top_disliked_genres": [{ "genre": "Horror", "count": 2 }],
  "recent_moods":   ["mood14", "mood13", "..."],
  "recent_queries": ["newest", "..."],
  "content_prefs":  { "no_horror": true }
}
```

**curl:**
```bash
curl "http://localhost:8000/me?session_id=demo:1"
```

### `DELETE /me`

Wipes per-session memory + conversation rows. **Irreversible.**

**Query parameters:**
- `session_id` (required)

**Response (200):**
```json
{
  "session_id": "demo:1",
  "deleted": { "memory_rows": 5, "conversation_rows": 1 }
}
```

**curl:**
```bash
curl -X DELETE "http://localhost:8000/me?session_id=demo:1"
```

### `POST /playlist`

Returns an N-track curated playlist for a mood. Two LLM calls
(profiler + playlist). Rate-limited and cost-capped.

**Request body** (`PlaylistRequest`):
```json
{
  "query": "string, 1-1000",
  "session_id": "string, 1-128",
  "length": 5
}
```
`length` is optional, default 5, must be 3–15.

**Response (200):** `Playlist` schema:
```json
{
  "mood_detected": "string",
  "title": "string (3-6 evocative words)",
  "intro": "string (1-2 sentences)",
  "tracks": [
    {
      "spotify_uri": "string",
      "track": "string",
      "artist": "string",
      "album": "string|null",
      "spotify_url": "string",
      "album_art_url": "string|null",
      "preview_url": "string|null",
      "reason": "string (1 sentence)"
    }
  ]
}
```

**Errors:**
- `422` — `length` out of range (`ge=3, le=15`)
- `429` — daily cap reached (`Demo limit reached for today — try again tomorrow.`)
- `500` — playlist build failed (Gemini error not handled by fallback)

**curl:**
```bash
curl -X POST http://localhost:8000/playlist \
  -H "Content-Type: application/json" \
  -d '{"query":"rainy sunday afternoon","session_id":"demo:1","length":5}'
```

### `POST /share`

Creates a public, anonymously-readable snapshot of a `Pairing`.
Returns an 8-character base32 short code (alphabet excludes
confusables: `0`, `o`, `1`, `l`, `i`).

**Request body** (`ShareRequest`):
```json
{
  "pairing": { "movie": "...", "music": "...", "pairing_note": "..." },
  "mood":    "string, 1-200"
}
```

**Response (200):** `ShareResponse`:
```json
{
  "short_code": "abc23xyz",
  "pairing":    { "movie": "...", "music": "...", "pairing_note": "..." },
  "mood":       "..."
}
```

**Errors:** `500` if all 5 short-code collision retries fail.

**curl:**
```bash
curl -X POST http://localhost:8000/share \
  -H "Content-Type: application/json" \
  -d '{"pairing": {"movie":"...","music":"...","pairing_note":"..."}, "mood": "reflective, cinematic"}'
```

---

### `GET /share/{short_code}`

Reads a previously-shared pairing. No auth. Used by the frontend's
`/p/[code]` public page.

**Path parameter:**
- `short_code` (required) — 4–16 chars, matching `^[a-z2-9]+$`

**Response (200):** Same `ShareResponse` shape as `POST /share`.

**Errors:**
- `404` — code not found
- `422` — code fails the regex (uppercase, special chars, etc.)

**curl:**
```bash
curl http://localhost:8000/share/abc23xyz
```
